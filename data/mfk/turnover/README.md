# Turnover JSON Specification

This directory contains derived datasets generated by `scripts/generate_turnover_json.py`. All monetary values represent turnover in euros and are encoded as JSON numbers (floating point). Counts such as taxpayers are JSON integers.

## `monthly_category_city_last_year.json`
- **Type:** Object  
- **Fields:**
  - `year` (`number`): Calendar year represented across records.
  - `records` (`array`): Monthly breakdown entries.
    - `month` (`number`): Month number `1-12`.
    - `category` (`string`): Sector description.
    - `city` (`string`): Municipality name.
    - `turnover` (`number`): Sum of turnover for the month/category/city.
    - `taxpayers` (`number`): Total taxpayers contributing to the turnover.

## `categories_last_year.json`
- **Type:** Object  
- **Fields:**
  - `year` (`number`): Calendar year represented across records.
  - `records` (`array`): Aggregated totals per category for `year`.
    - `category` (`string`)
    - `turnover` (`number`)
    - `taxpayers` (`number`)

## `cities_last_year.json`
- **Type:** Object  
- **Fields:**
  - `year` (`number`): Calendar year represented across records.
  - `records` (`array`): Aggregated totals per city for `year`.
    - `city` (`string`)
    - `turnover` (`number`)
    - `taxpayers` (`number`)

## `categories_over_years.json`
- **Type:** Array  
- **Item Fields:**
  - `year` (`number`)
  - `category` (`string`)
  - `turnover` (`number`)
  - `taxpayers` (`number`)

## `top_category_by_city_over_years.json`
- **Type:** Array  
- **Description:** Up to the top eight categories per city/year, ordered by turnover.
- **Item Fields:**
  - `year` (`number`)
  - `city` (`string`)
  - `rank` (`number`): Rank of the category within the city/year, `1` is highest turnover.
  - `category` (`string`): Category corresponding to the rank.
  - `turnover` (`number`): Turnover for the ranked category.
  - `taxpayers` (`number`): Taxpayer count for the ranked category.

### Notes
- Files use UTF-8 encoding without BOM.
- Keys remain stable; new files will append or extend record sets without restructuring existing fields.
